# Citation Integrity Protocol (CIP v1.0)

This protocol ensures that all academic submissions generated by the AI Mastermind Alliance are free from hallucinated or "suspicious" citations.

## The Absolute Rule

**No citation shall be entered into a final document unless it has been explicitly verified against a live academic database (OpenAlex, Semantic Scholar, Google Scholar, or DFO CSAS).**

## 1. Discovery Phase (@Comet / Browser)

- Use search queries that prioritize DOIs and official publication records.
- **Mandatory Fields**: Title, Full Author List, Year, Journal Name, Volume, Issue, Page Range/Article ID, and DOI.
- If a DOI is not available, the source must be flagged as "low-confidence" and avoided if possible.

## 2. Verification Phase (@Antigravity)

- Use `run_command` to ping the DOI or search for the specific Volume/Year correlation.
- **Hallucination Red Flags**:
  - Volume numbers that don't match the journal's age (e.g., Vol 250 in 2023 for a 50-year-old journal).
  - Generic titles that perfectly match the prompt (e.g., "The Problem with Salmon in Moncton").
  - Article IDs (`e1234`) used in journals that typically use page numbers.

## 3. Integration Phase (@Architect)

- Cite only verified sources.
- When summarizing, use specific details from the abstract to ensure the citation is contextually accurate, not just a label.

## 4. Final Audit

- Use `textutil` or similar tools to scan for formatting markers (`*`, `**`) that indicate a raw AI export.
- Conduct a "Voice Pass" to break up overly-perfect sentence structures and use a hybrid of academic and personal student-voice transitions.
