# AILCC Second Brain: Operational Protocols 🧠

The AILCC system utilizes a **PARA** (Projects, Areas, Resources, Archives) structure and a **CODE** (Capture, Organize, Distill, Express) workflow to ensure absolute cognitive clarity and agentic efficiency.

## 📂 PARA Structure

### **00_Projects**
- **Definition**: Active tasks with a specific deadline and objective.
- **Rule**: If a folder has no active Linear ticket or `task.md`, it must be moved to Areas or Archives.
- **Examples**: `Mission_Automation_Hub`, `Orchestration_Hub`.

### **01_Areas**
- **Definition**: Ongoing responsibilities that require a standard of performance over time.
- **Sub-folders**:
  - `Neuroscience`: MSc academic track.
  - `ECFC`: Business operations (East Coast Fresh Coats).
  - `Codebases`: The long-term evolution of the AILCC system.
  - `Hippocampus`: Long-term memory and vector stores.

### **02_Resources**
- **Definition**: Topics of ongoing interest or reference material.
- **Examples**: `Habits`, `Templates`, `Protocols`.

### **03_Archives**
- **Definition**: Completed or inactive items from the other three categories.

---

## ⚡ CODE Workflow

### **1. Capture**
- **Agents**: Comet (Browser), Whisper (Voice), Mobile (iOS Shortcuts).
- **Protocol**: All new data lands in `07_Comet_Sync` or `04_Intelligence_Vault` tagged as `UNPROCESSED`.

### **2. Organize**
- **Agents**: Sentinel / Orator.
- **Protocol**: Move captured data into the appropriate PARA folder within 24 hours.

### **3. Distill**
- **Agents**: Grok / Anti-Gravity.
- **Protocol**: Extract the "essence" of documents into Knowledge Items (KIs). Update the Brain with summaries.

### **4. Express**
- **Agents**: Forge / Archon.
- **Protocol**: Generate outputs (Thesis chapters, Invoices, Diplomatic Dispatches) using distilled knowledge.

## 🛡️ Governance
- **The Architect**: Periodic review of PARA alignment.
- **The Judge**: Automated "Entropy Scan" to identify misfiled assets.
